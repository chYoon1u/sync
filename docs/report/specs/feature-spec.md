## 3. Spotify 음악 플레이어

> 전제: Spotify Premium 계정 필요. 인증 방식은 PKCE OAuth이며 백엔드는 사용하지 않는다. 개발 모드 앱은 앱 소유자의 Premium 구독과 허용 사용자 등록이 필요하다.

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| P-01 | Spotify 로그인 | `127.0.0.1` loopback Redirect URI와 PKCE `state` 검증을 사용하고 access token / refresh token 및 로그인 오류를 관리 | High |
| P-02 | 곡 검색 | 제목 또는 아티스트로 검색하고 개발 모드 API 상한에 맞춰 결과 10개를 표시 | High |
| P-03 | 바로 재생 | 검색 결과에서 즉시 재생 | High |
| P-04 | 곡 추가 모달 | `곡 추가` 버튼을 눌렀을 때만 별도 모달에서 검색 후 플레이리스트에 곡 추가 | High |
| P-05 | 재생 / 일시정지 | Web Playback SDK `togglePlay` 사용 | High |
| P-06 | 다음 / 이전 곡 | 순환 재생, 3초 이상 재생 중이면 이전 곡 대신 `seek(0)` | High |
| P-07 | 볼륨 조절 | 슬라이더 0~100, 음소거 토글 포함 | High |
| P-08 | 진행 바 | 하단 PlayerBar 전체 가로 폭에 맞춘 진행 바와 클릭/드래그 seek 지원 | High |
| P-09 | 반복 재생 | 트랙 반복 모드 제어 | Medium |
| P-10 | 셔플 | 셔플 모드 제어 | Medium |
| P-11 | 현재 재생곡 표시 | 화면 하단 고정 PlayerBar에서 앨범아트, 제목, 아티스트, 진행 상태 표시 | High |
| P-12 | 플레이리스트 목록 팝업 | 기본적으로 `플레이리스트` 버튼 클릭 시 작은 팝업 목록을 열고 스크롤 가능하게 표시하며 스크롤바는 숨김 | High |
| P-13 | 재생 이력 복원 | 마지막 재생 곡과 재생 위치를 localStorage에 저장하고 앱 재실행 시 자동 복원 재생 | High |
| P-14 | 전역 상태 유지 | PlayerBar는 App 루트에서 한 번만 마운트해 화면 전환과 무관하게 유지 | High |
| P-15 | 토큰 자동 갱신 | 만료 5분 전 refresh token으로 access token 자동 갱신 | High |
